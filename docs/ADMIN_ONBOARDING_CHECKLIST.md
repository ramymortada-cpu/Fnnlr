# Admin Onboarding Checklist

Use this for every customer setup. Do not mark a step done without evidence.

| Step | Evidence |
| --- | --- |
| Industry selected | onboarding answer or config |
| Goal selected | onboarding answer or config |
| Owner user created | customer setup output |
| Business created | tenant/business id |
| First template selected | template id |
| First workflow/page created | workflow/page id |
| First workflow/page published | publish output |
| First tracked link created | tracked link id |
| First smoke signal recorded | smoke event id |
| First support owner assigned | support handoff |
| First daily check scheduled | operating room output |

## Failure Recovery

If onboarding stops:

1. Record last completed step.
2. Assign owner.
3. Record blocker type: credentials, content, domain, payment instructions, customer owner, or product bug.
4. Set next action and due date.
5. Re-run activation status after fix.
