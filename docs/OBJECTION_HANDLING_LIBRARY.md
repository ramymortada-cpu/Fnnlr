# Objection Handling Library

| Objection | Safe response |
| --- | --- |
| Can it guarantee sales? | No. It improves visibility, follow-up, and evidence-based action, but revenue depends on offer, traffic, team response, and market. |
| Does it send WhatsApp automatically? | No. It drafts and recommends; a human approves and sends. |
| Does it process payments? | No. GA v1 records payment state and follow-up needs; it does not move money. |
| Is it secure enough? | Tenant isolation, secret encryption, MFA/admin controls, and audit evidence are implemented; hosted proof is required before GA. |
| Why not use a normal CRM? | fnnlr focuses on Arabic-first revenue workflows, WhatsApp, manual payment states, and evidence-based operating actions. |
| Is it self-serve? | Initial launch is assisted; self-serve packaging is on the roadmap after hosted evidence and onboarding metrics. |
